# AI Harness Engineer 完整逐场景旁白稿

**日期**: 2026-03-28  
**用途**: 直接进入配音、字幕同步与 composition 制作  
**建议场景数**: 7  
**建议总时长**: 100-115 秒  
**语气**: 强判断、轻危机、重系统感  
**表达边界**: 不把 `AI Harness Engineer` 说成已经统一定型的正式岗位名称，而是强调“角色标签还在形成，但对应工作已经真实存在”。

## 使用说明

- 每段旁白对应一个场景
- 语速按中等偏快处理
- 若进入 TTS，可保留少量 `...` 做自然停顿
- 核心判断尽量落在句尾，方便字幕切句和画面卡点

## Scene 1

**标题**: Prompt 不够了  
**目标**: 用冲突开场，快速打破旧认知  
**建议时长**: 12-14 秒

**旁白**:

你以为 AI 时代最值钱的是提示词吗？... 但 2026 年开始冒头的，可能已经不是 Prompt Engineer。越来越值钱的，是能让代理稳定干活的人。

## Scene 2

**标题**: Demo 能跑，生产跑不动  
**目标**: 把观众从概念拉回真实痛点  
**建议时长**: 13-15 秒

**旁白**:

很多 agent demo 看起来都很强，可一进真实环境就出问题。权限不清楚，上下文漂移，失败没法回放，行为也看不见。企业真正缺的，不是会演示的 AI，而是可靠代理。

## Scene 3

**标题**: 什么是 Harness  
**目标**: 给出可传播定义  
**建议时长**: 13-15 秒

**旁白**:

所以什么叫 Harness？... 它不是单个框架，也不只是多接几个工具。Harness 是代理外面的整套运行系统，负责上下文、记忆、权限、评测、观测和执行环境。

## Scene 4

**标题**: OpenAI 已经下场  
**目标**: 给出最强一手信号  
**建议时长**: 14-16 秒

**旁白**:

这不是社区自嗨。OpenAI 已经公开在讲 Harness Engineering，还把重点放在设计环境、指定意图和反馈回路。连 Codex 的 App Server 也在说明，真正重要的，已经是代理运行外壳。

## Scene 5

**标题**: 能力已经被拆开  
**目标**: 说明这个方向已经进入工程模块化  
**建议时长**: 13-15 秒

**旁白**:

再看 LangChain 这类文档，harness 的能力已经被拆开了。规划、文件系统、子代理、执行、人类审批、skills、memory，全都在模块化。MCP 也在强调边界和用户同意。行业正在从怎么提问，走到怎么把代理系统搭对。

## Scene 6

**标题**: 谁会先吃到红利  
**目标**: 把趋势翻译成职业机会  
**建议时长**: 13-15 秒

**旁白**:

所以未来更值钱的，不只是会写 prompt 的人。真正吃到红利的，是能让代理稳定交付结果的人。如果你已经在做 workflow、sandbox、evals、guardrails、observability，你其实已经在往这个方向走了。

## Scene 7

**标题**: 现在就该补的能力  
**目标**: 收束观点并给出行动建议  
**建议时长**: 12-14 秒

**旁白**:

所以别再只盯着花哨 demo 了。下一阶段真正稀缺的，是把 AI 变成可靠系统。少一点提示词神话，多补运行环境、权限控制、评测和可见性。

## 进入制作前的核对项

- 旁白主张与策划稿一致
- 每段只承担一个核心功能
- 开头有冲突，中段有证据，结尾有判断
- 避免把趋势说成已经完全定型的行业事实
- 可以直接进入配音脚本与字幕同步阶段
